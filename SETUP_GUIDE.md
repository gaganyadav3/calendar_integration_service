# Calendar Service - Setup Guide

## Development Environment Setup

### 1. Prerequisites Installation

#### Java 21
```bash

# macOS (using Homebrew)
brew install openjdk@21

# Verify installation

```

#### Maven 3.8+
```bash
# Windows (using Chocolatey)
choco install maven

# macOS (using Homebrew)
brew install maven

# Verify installation
mvn -version
```

#### Oracle Database (or use H2 for development)
For development, you can use H2 in-memory database. For production, set up Oracle Database 19c or later.

### 2. OAuth Provider Configuration

#### Google Cloud Setup
1. Visit [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project named "Calendar Service"
3. Navigate to "APIs & Services" → "Credentials"
4. Create OAuth 2.0 Web Application credentials
5. Add these redirect URIs:
   - `http://localhost:8090/api/oauth/callback/google`
   - `http://localhost:3000/callback` (for frontend)
6. Download the credentials JSON
7. Copy `Client ID` and `Client Secret`

#### Microsoft Azure Setup
1. Visit [Azure Portal](https://portal.azure.com)
2. Go to "Azure Active Directory" → "App registrations"
3. Click "New registration"
4. Name: "Calendar Service"
5. Supported account types: "Accounts in any organizational directory"
6. Redirect URI: `http://localhost:8090/api/oauth/callback/outlook`
7. Go to "Certificates & secrets" and create a new client secret
8. Go to "API permissions" and add:
   - `Calendars.ReadWrite`
   - `Calendars.Read`
   - `offline_access`
9. Copy `Application (client) ID` and `Client secret value`

```properties
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

# Alternative: H2 Database (for development)
# spring.datasource.url=jdbc:h2:mem:omvrti-db
# spring.datasource.driver-class-name=org.h2.Driver
# spring.datasource.username=sa
# spring.datasource.password=
# spring.h2.console.enabled=true

# Google OAuth Configuration

oauth.google.client-secret=YOUR_GOOGLE_CLIENT_SECRET

# Logging
logging.level.root=INFO
logging.level.com.omvrti.calendar_service=DEBUG
logging.pattern.console=%d{yyyy-MM-dd HH:mm:ss} - %msg%n
```

### 5. Database Setup

#### For Oracle Database
```sql
-- The application will auto-create tables with `spring.jpa.hibernate.ddl-auto=update`
-- If you need manual setup, create these tables:

CREATE TABLE OAUTH_TOKENS (
  id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_email VARCHAR2(255) NOT NULL,
  provider VARCHAR2(50) NOT NULL,
  access_token CLOB NOT NULL,
  refresh_token CLOB,
  expires_in NUMBER,
  token_type VARCHAR2(50),
  scope CLOB,
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  refreshed_at TIMESTAMP,
  UNIQUE (user_email, provider)
);

CREATE TABLE CALENDAR_EVENTS (
  id VARCHAR2(255) PRIMARY KEY,
  user_email VARCHAR2(255) NOT NULL,
  provider VARCHAR2(50),
  external_id VARCHAR2(255),
  summary VARCHAR2(500),
  description CLOB,
  location VARCHAR2(255),
  start_date_time TIMESTAMP,
  end_date_time TIMESTAMP,
  start_date DATE,
  end_date DATE,
  status VARCHAR2(50),
  organizer VARCHAR2(255),
  is_all_day CHAR(1) DEFAULT 'N',
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

### 8. Development Tips

#### Hot Reload (Spring Boot DevTools)
Add to `pom.xml`:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <scope>runtime</scope>
    <optional>true</optional>
</dependency>
```

#### Debug Mode
```bash
# Unix/Linux/macOS
mvn spring-boot:run -Dspring-boot.run.jvmArguments="-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005"

# Windows
mvn spring-boot:run -Dspring-boot.run.jvmArguments="-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005"
```

Then connect debugger to `localhost:5005`

#### Test HTTP Requests
Use Postman or VS Code REST Client to test endpoints. Example:
```bash
# .http file for VS Code REST Client
@baseUrl = http://localhost:8090
@userEmail = test@example.com

### Get authorization URL
GET {{baseUrl}}/api/oauth/authorize?provider=GOOGLE

### Exchange code for token
POST {{baseUrl}}/api/oauth/token/exchange
Content-Type: application/json

{
  "code": "authorization_code",
  "provider": "GOOGLE",
  "userEmail": "{{userEmail}}"
}

### Get events
GET {{baseUrl}}/api/calendar/events
X-USER-EMAIL: {{userEmail}}
```

### 9. Troubleshooting

| Issue | Solution |
|-------|----------|
| `java.lang.UnsupportedClassVersionError` | Update Java to version 21 |
| `Connection refused to database` | Verify database URL and credentials, ensure database is running |
| `401 Unauthorized` | Re-authenticate through OAuth flow, token may have expired |
| `Maven build fails` | Run `mvn clean` and `mvn install` again |
| `Port 8090 already in use` | Change port in `application.properties` or kill process using port |
| `No qualifying bean of type 'IOAuthProvider'` | Check `ProvidersConfiguration.java` and ensure providers are registered |
